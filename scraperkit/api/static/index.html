📖 Guide & Config Reference

Quick Start

Install ScraperKit, create a YAML config file, and run it in one command.

1. Install

pip install -e ".[all]"

2. Create a config file

name: my_project

start_urls:
  - https://books.toscrape.com/catalogue/page-1.html

crawler:
  item_selector: "article.product_pod"
  fields:
    title: "h3 a::attr(title)"
    price: "p.price_color::text"
    url:   "h3 a::attr(href)"
  pagination:
    type: css
    selector: "li.next a::attr(href)"

compare:
  key_field: url

workflow:
  - crawl
  - clean
  - deduplicate
  - export_json
  - export_excel
  - compare_previous

3. Run it

scraperkit run my_project.yaml

Output appears in output/my_project/json/ and output/my_project/excel/.

4. Start the admin panel

scraperkit serve          # http://localhost:8000
scraperkit serve --port 9000 --host 0.0.0.0
Config — Top-Level Fields
FieldTypeDescription
namestringProject name. Used for output folder. Auto-inferred from filename if omitted.
start_urlslistOne or more URLs the spider starts from.
crawlerobjectSpider and extraction settings (see below).
workflowlistOrdered list of step names to run. Default: [crawl, export_json]
compareobjectChange detection settings.
hooksobjectEvent-triggered actions.
outputobjectOutput directory settings.
notifyobjectSlack / email / SharePoint.
extraobjectFree-form data accessible in custom steps via ctx.config.extra.
crawler — HTML Sites 
crawler:
  item_selector: ".product-card"   # CSS selector — one match = one item
  fields:
    title: ".title::text"           # shorthand CSS
    price: ".price::text"
    link:  "a::attr(href)"
    name:  "xpath=//h1/text()"      # XPath prefix
    sku:   "regex=SKU-(\d+)"        # Regex prefix — group 1 returned
  pagination:
    type: css                        # css | xpath | url_increment
    selector: "a.next::attr(href)"
  delay_min: 2.0                     # seconds between requests
  delay_max: 5.0
  autothrottle: true
  rotate_useragent: true
  respect_robots: true

Field shorthand prefixes

SyntaxTypeExample
.selector::textCSS.title::text
xpath=...XPathxpath=//h1/text()
regex=...Regexregex=price:\s*(\d+)

Full field form

fields:
  price:
    type: css           # css | xpath | regex | json
    selector: ".price::text"
    default: "N/A"
    transform: strip    # strip | lower | upper | int | float
crawler — JSON APIs

When the URL returns JSON instead of HTML, set response_type: json.

start_urls:
  - https://api.example.com/dealers?page=1&limit=100

crawler:
  response_type: json
  json_items_path: "items"          # dot-path to the array in the JSON response
  json_total_pages_path: "total_pages"  # optional — stops pagination early

  # Leave fields empty to pass through ALL keys from the API item.
  # Or list only the fields you need:
  fields:
    id:   "id"
    name: "name"
    city: "city"
    zip:  "zip"

  pagination:
    type: url_increment
    param: page               # query param to increment
    # max_pages: 50           # hard stop (optional)

  respect_robots: false       # API endpoints don't have robots.txt

Pagination stops automatically when the items array is empty, or when total_pages is reached.

Nested paths

json_items_path: "data.results"   # navigates response["data"]["results"]

fields:
  address: "location.address"       # navigates item["location"]["address"]
Pagination Types

CSS / XPath — follow a "next page" link

pagination:
  type: css
  selector: "a.next-page::attr(href)"

URL increment — bump ?page=N

pagination:
  type: url_increment
  param: page          # query parameter name
  max_pages: 50        # optional hard limit
Workflow Steps
StepWhat it does
crawlRun the Scrapy spider. Loads items into the pipeline.
cleanStrip whitespace, remove items where all fields are empty/null.
deduplicateRemove duplicate items using compare.key_field.
export_jsonWrite output/<project>/json/<ts>_items.json
export_excelWrite output/<project>/excel/<ts>_items.xlsx
compare_previousDetect new / removed / changed items vs the last run.
backupArchive all output files to a timestamped folder.
notify_slackPost summary to Slack. Needs SLACK_TOKEN env var.
notify_emailSend summary email via SMTP.
upload_sharepointUpload files to SharePoint via Microsoft Graph.
 
workflow:
  - crawl
  - clean
  - deduplicate
  - export_json
  - export_excel
  - compare_previous
  - backup
  # - notify_slack      # uncomment + set SLACK_TOKEN env var
Change Detection (compare_previous)

After crawling, compare with the previous run to detect what's new, removed, or changed.

compare:
  key_field: id            # unique field to identify each item
  fuzzy_fields:            # fields to fuzzy-match when key doesn't match
    - name
    - address
  fuzzy_threshold: 97      # fuzzywuzzy token_set_ratio (0–100)

Results are saved to output/<project>/compare/<ts>_comparison.json and also available in ctx.meta["compare"] for hook steps like notify_slack.

Output structure

{
  "new":     [...],   # items in current run, not in previous
  "removed": [...],   # items in previous run, not in current
  "changed": [...],   # same key, different field values
  "fuzzy_matches": [...],
  "counts":  { "new": 5, "removed": 2, "changed": 8, ... }
}
Hooks

Hooks fire automatically on lifecycle events. Each handler can be a step name, a shell command, or a Python function.

hooks:
  on_start:           []
  on_step_success:    []
  on_step_error:      []
  on_crawl_finished:  []
  on_new_items_found: []
  on_workflow_failed:
    - notify_slack              # run a registered step
    - shell:echo "failed"       # shell command
  on_workflow_success:
    - upload_sharepoint
EventWhen it fires
on_startBefore any step runs
on_step_successAfter each successful step
on_step_errorAfter a step fails
on_crawl_finishedAfter the crawl step, regardless of item count
on_new_items_foundAfter crawl, only if items > 0
on_workflow_failedIf any step errors out
on_workflow_successAll steps completed successfully
Notifications — Slack, Email, SharePoint

All credentials are read from environment variables. Never put secrets in the YAML file.

Slack

notify:
  slack:
    token_env: SLACK_TOKEN     # name of the env var holding your bot token
    channel: "#my-channel"
    test_channel: "#my-dev"    # optional separate test channel

# Then set the env var:
# export SLACK_TOKEN=xoxb-your-token

Email (SMTP)

notify:
  email:
    smtp_host: smtp.gmail.com
    smtp_port: 587
    from_addr: me@example.com
    to_addrs: [you@example.com]
    username_env: SMTP_USER
    password_env: SMTP_PASS

SharePoint (Microsoft Graph)

notify:
  sharepoint:
    tenant_id_env:     SP_TENANT_ID
    client_id_env:     SP_CLIENT_ID
    client_secret_env: SP_CLIENT_SECRET
    site_url: "https://tenant.sharepoint.com/sites/MySite"
    drive_id: "YOUR_DRIVE_ID"
    remote_folder: "General/reports"
    keep_only_latest: false   # true = replace old files
Writing a Custom Step 
from scraperkit.core.base import BaseStep
from scraperkit.core.context import RunContext
from scraperkit.core.registry import register_step

@register_step("my_step")
class MyStep(BaseStep):
    def execute(self, ctx: RunContext) -> dict:
        # ctx.items  — current items list (modify in-place)
        # ctx.config — full ProjectConfig
        # ctx.meta   — shared dict between steps
        # ctx.output_dir — pathlib.Path to output folder

        for item in ctx.items:
            item["enriched"] = True

        return {"enriched": len(ctx.items)}

Register it before running:

import my_module.steps.my_step   # triggers @register_step
from scraperkit.core.config import load_config
from scraperkit.core.runner import WorkflowRunner
WorkflowRunner(load_config("project.yaml")).run()

Then use it in your YAML:

workflow:
  - crawl
  - my_step
  - export_json
CLI Reference 
scraperkit run <config.yaml>            # run a workflow
scraperkit run <config.yaml> --dry-run  # validate config, don't execute
scraperkit run <config.yaml> -o ./out   # custom output directory
scraperkit run <config.yaml> -l DEBUG   # verbose logging

scraperkit serve                         # admin panel at localhost:8000
scraperkit serve --host 0.0.0.0 --port 9000

scraperkit runs                          # table of recent runs
scraperkit runs --project my_project     # filter by project
scraperkit show <run_id>                 # step-by-step detail
